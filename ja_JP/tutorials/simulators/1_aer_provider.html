シミュレーター¶

はじめに¶

このノートブックでは、*Qiskit Aer*シミュレーターのバックエンドをインポートし、理想的な(ノイズフリーの) Qiskit Terra回路を実行する方法を説明します。

[1]:

import numpy as np

# Import Qiskit
from qiskit import QuantumCircuit, QuantumRegister, ClassicalRegister
from qiskit import Aer, execute
from qiskit.tools.visualization import plot_histogram, plot_state_city

Qiskit Aer シミュレーターのバックエンド¶

Qiskit Aerには現在3つの高性能シミュレーターバックエンドが含まれています: * QasmSimulator : Qiskit回路の理想的でノイズの多発的な実行を可能にし、カウントまたはメモリを返します* StatevectorSimulator : qiskit回路の理想的なシングルショット実行を可能にし、アプリケーションの後にシミュレーターの最終状態ベクトルを返します* UnitarySimulator : qiskit回路の理想的なシングルショット実行を可能にし、回路自体の最終的なユニタリー行列を返します。 このバックエンドの測定またはリセット操作を回路に含めることはできないことに注意してください

これらのバックエンドは qasm_simulator と statevector_simulator と unitary_simulator という名前の Aer プロバイダにあります。

[2]:

# List Aer backends
Aer.backends()

[2]:

[<QasmSimulator('qasm_simulator') from AerProvider()>,
 <StatevectorSimulator('statevector_simulator') from AerProvider()>,
 <UnitarySimulator('unitary_simulator') from AerProvider()>,
 <PulseSimulator('pulse_simulator') from AerProvider()>]

シミュレーターのバックエンドは qiskit.providers.aer から直接インポートすることもできます。

[3]:

from qiskit.providers.aer import QasmSimulator, StatevectorSimulator, UnitarySimulator

QasmSimulator¶

QasmSimulator バックエンドは実際のデバイスを模倣するように設計されています。 QisKit QuantumCircuit を実行し、回路内のすべての古典レジスタの最終値を含むカウント辞書を返します。 回路には、別のノートブックで説明する、ゲート、測定、リセット、条件、およびその他の高度なシミュレーターオプションが含まれている場合があります。

量子回路のシミュレーション¶

基本操作は、量子回路を実行し測定結果のカウント・ディクショナリを返却することです。ここでは、2量子ビットのベル状態 \(|\psi\rangle = \frac{1}{2}(|0,0\rangle + |1,1 \rangle)\) を作成する簡単な回路を実行し、両方の量子ビットを測定します。

[4]:

# Construct quantum circuit
circ = QuantumCircuit(2, 2)
circ.h(0)
circ.cx(0, 1)
circ.measure([0,1], [0,1])

# Select the QasmSimulator from the Aer provider
simulator = Aer.get_backend('qasm_simulator')

# Execute and get counts
result = execute(circ, simulator).result()
counts = result.get_counts(circ)
plot_histogram(counts, title='Bell-State counts')

[4]:

各ショットの測定結果を返します¶

QasmSimulator では、各ショットの測定結果のリストを返すこともできます。 これは、assemble または execute 関数でキーワード引数 memory=True を設定することで有効になります。

[5]:

# Construct quantum circuit
circ = QuantumCircuit(2, 2)
circ.h(0)
circ.cx(0, 1)
circ.measure([0,1], [0,1])

# Select the QasmSimulator from the Aer provider
simulator = Aer.get_backend('qasm_simulator')

# Execute and get memory
result = execute(circ, simulator, shots=10, memory=True).result()
memory = result.get_memory(circ)
print(memory)

['00', '00', '00', '11', '11', '00', '00', '11', '11', '00']

カスタム初期状態でシミュレーションを開始¶

QasmSimulator では、シミュレーションにおけるカスタム初期状態ベクトルを設定することができます。これは、Qobjにおけるすべての実験が、すべてゼロの状態 \(|0,0,..0\rangle\) からではなく、\(|\psi\rangle\) 状態から開始できることを意味します。カスタム状態は``initialize``メソッドを使って回路に設定できます。

注意: * 初期状態ベクトルは有効な量子状態 \(|\langle\psi|\psi\rangle|=1\) でなければなりません。そうでない場合は、例外が発生します。 * シミュレーターはこのオプションを直接サポートしますが、実際のデバイスで実行するための標準ゲートにアンロールすることもできます。

ここでは、前述の例の最後のBell-state でシミュレーターを初期化することによって、この機能を示します。

[6]:

# Construct a quantum circuit that initialises qubits to a custom state
circ = QuantumCircuit(2, 2)
circ.initialize([1, 0, 0, 1] / np.sqrt(2), [0, 1])
circ.measure([0,1], [0,1])

# Select the QasmSimulator from the Aer provider
simulator = Aer.get_backend('qasm_simulator')

# Execute and get counts
result = execute(circ, simulator).result()
counts = result.get_counts(circ)
plot_histogram(counts, title="Bell initial statevector")

[6]:

StatevectorSimulator¶

StatevectorSimulator は、Qiskit QuantumCircuit の単一ショットを実行し、シミュレーションの最終的な量子状態ベクトルを返します。 回路には*ゲート*、測定、リセット、および*条件*の操作を含めることができます。

量子回路のシミュレーション¶

基本操作は、量子回路を実行し測定結果のカウント・ディクショナリを返却することです。ここでは、2量子ビットのベル状態 \(|\psi\rangle = \frac{1}{2}(|0,0\rangle + |1,1 \rangle)\) を作成する簡単な回路を実行し、両方の量子ビットを測定します。

[7]:

# Construct quantum circuit without measure
circ = QuantumCircuit(2)
circ.h(0)
circ.cx(0, 1)

# Select the StatevectorSimulator from the Aer provider
simulator = Aer.get_backend('statevector_simulator')

# Execute and get counts
result = execute(circ, simulator).result()
statevector = result.get_statevector(circ)
plot_state_city(statevector, title='Bell state')

[7]:

測定による量子回路のシミュレーション¶

回路に*measure*または*reset*が含まれている場合、最終状態ベクトルは、波動関数の崩壊を測定またはリセットの結果にシミュレーションする*後*の条件状態ベクトルになります。 ベル状態回路の場合、これは最終的な状態ベクトルは * \(|0,0\rangle\) または \(|1, 1\rangle\) であることを意味します。

[8]:

# Construct quantum circuit with measure
circ = QuantumCircuit(2, 2)
circ.h(0)
circ.cx(0, 1)
circ.measure([0,1], [0,1])

# Select the StatevectorSimulator from the Aer provider
simulator = Aer.get_backend('statevector_simulator')

# Execute and get counts
result = execute(circ, simulator).result()
statevector = result.get_statevector(circ)
plot_state_city(statevector, title='Bell state post-measurement')

[8]:

カスタム初期状態でシミュレーションを開始¶

QasmSimulator と同様に、StatevectorSimulator もシミュレーションにカスタムの初期状態ベクトルを設定することができます。 ここでは、StatevectorSimulator 上で以前の初期状態ベクトルの例を実行し、Bell 状態に初期化します。

[9]:

# Construct a quantum circuit that initialises qubits to a custom state
circ = QuantumCircuit(2)
circ.initialize([1, 0, 0, 1] / np.sqrt(2), [0, 1])

# Select the StatevectorSimulator from the Aer provider
simulator = Aer.get_backend('statevector_simulator')

# Execute and get counts
result = execute(circ, simulator).result()
statevector = result.get_statevector(circ)
plot_state_city(statevector, title="Bell initial statevector")

[9]:

Unitary Simulator¶

UnitarySimulator は、それぞれのゲート行列をアイデンティティ行列に適用することで、Qiskit QuantumCircuit のユニタリー行列を構築します。 回路には*ゲート*が含まれている場合、*resets*または*measure*演算が含まれている場合にのみ、例外が発生します。

量子回路のユニタリーをシミュレートする¶

この例では、ベル状態を準備する以前の例の回路に対応するユニタリー行列を返します。

[10]:

# Construct an empty quantum circuit
circ = QuantumCircuit(2)
circ.h(0)
circ.cx(0, 1)

# Select the UnitarySimulator from the Aer provider
simulator = Aer.get_backend('unitary_simulator')

# Execute and get counts
result = execute(circ, simulator).result()
unitary = result.get_unitary(circ)
print("Circuit unitary:\n", unitary)

カスタム初期ユニタリーの設定¶

``UnitarySimulator``の初期状態を設定することもできますが、この状態は初期の*ユニタリ行列* \(U_i\) であり、状態ベクトルではありません。この場合に返されるユニタリは、最初のユニタリ行列に対してユニタリ回路を適用することにより与えられる \(U.U_i\) になります。

注意: * 初期ユニタリーは有効なユニタリー行列 \(U^\dagger.U =\mathbb{1}\) に複数の実験が含まれている場合、 初期ユニタリーは Qobj での すべての 実験のための正しいサイズでなければなりません。そうしないと例外が発生します。

シミュレーターの初期状態として、以前の回路の出力ユニタリを準備しましょう。

[13]:

# Construct an empty quantum circuit
circ = QuantumCircuit(2)
circ.id([0,1])

# Set the initial unitary
opts = {"initial_unitary": np.array([[ 1,  1,  0,  0],
                                     [ 0,  0,  1, -1],
                                     [ 0,  0,  1,  1],
                                     [ 1, -1,  0,  0]] / np.sqrt(2))}

# Select the UnitarySimulator from the Aer provider
simulator = Aer.get_backend('unitary_simulator')

# Execute and get counts
result = execute(circ, simulator, backend_options=opts).result()
unitary = result.get_unitary(circ)
print("Initial Unitary:\n", unitary)
