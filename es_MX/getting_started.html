Primeros Pasos con Qiskit¶

El flujo de trabajo con Qiskit consiste de 3 pasos principales:

  • Construcción: diseñar un circuito cuántico que represente el problema que está siendo considerando.

  • Ejecución: ejecutar experimentos en diferentes backends (los cuales incluyen tanto sistemas como simuladores).

  • Análisis: calcular resúmenes de estadísticas y visualizar resultados de los experimentos.

He aquí un ejemplo de todo el flujo de trabajo. Cada paso es explicado en detalle en las secciones subsecuentes:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = Aer.get_backend('qasm_simulator')

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# Execute the circuit on the qasm simulator
job = execute(circuit, simulator, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

Flujo de trabajo paso a paso¶

El programa presentado anteriormente puede ser dividido en seis pasos:

  1. Importar paquetes

  2. Inicializar variables

  3. Agregar compuertas

  4. Visualizar el circuito

  5. Simular el experimento

  6. Visualizar los resultados

Paso 1: Importar Paquetes¶

Los elementos básicos necesarios para el programa son importados como se muestra a continuación:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

Con más detalle, las importaciones son

  • QuantumCircuit: puede ser interpretado como las instrucciones del sistema cuántico. Contiene todas las operaciones cuánticas.

  • execute: ejecuta el circuito / experimento.

  • Aer: maneja los backends de simulación.

  • plot_histogram: crea histogramas.

Paso 2: Inicializar Variables¶

Considere la siguiente linea de código

circuit = QuantumCircuit(2, 2)

Aquí, estamos inicializando un circuito con 2 qubits en el estado cero; 2 bits clásicos con valor cero; y circuit es el circuito cuántico.

Sintaxis:

  • QuantumCircuit(int, int)

Paso 3: Agregar Compuertas¶

Podemos agregar compuertas (operaciones) para manipular los registros del circuito.

Considera las siguientes tres lineas de codigo:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

Las compuertas son agregadas al circuito una por una para formar el siguiente «Bell state»

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

El código mostrado anteriormente utiliza las siguientes compuertas:

  • QuantumCircuit.h(0): Una compuerta «Hadamard» :math: “H” en el qubit 0, que pone al qubit en un estado de superposición.

  • «QuantumCircuit.cx(0,1)»: Una operación controlled-Not (:math:”C_{X}”) en el cúbit de control 0 y el cúbit objetivo 1, poniendo a los cúbits en un estado de entrelazamiento.

  • QuantumCircuit.measure([0,1],[0,1]): Si pasa los registros cuánticos y clásicos a la instrucción measure, el resultado de la medición sobre el i-ésimo cúbit sera almacenado en el i-ésimo bit clásico.

Paso 4: Visualizar el Circuito¶

Puede usar QuantumCircuit.draw() para ver el circuito que hemos diseñado en los diferentes formatos usados en libros y artículos de investigación.

circuit.draw()

     ┌───┐     ┌─┐   
q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

En este circuito, los qubits son ordenados con el qubit cero en la parte superior y el qubit uno en la parte inferior. Este circuito se lee de izquierda a derecha, lo que quiere decir que, las compuertas se aplican de izquierda a derecha.

El «backend» que Qiskit usa por defecto para QuantumCircuit.draw() o qiskit.visualization.circuit_drawer() es el backend de texto. Sin embargo, dependiendo de nuestro entorno local, es posible cambiar estos valores predeterminados por otros más adecuados. Esto se hace con el archivo de configuración de usuario. Por defecto, el archivo de configuración de usuario se encuentra ubicado en ~/.qiskit/settings.conf y es un archivo .ini.

Por ejemplo, un archivo settings.conf para ajustar un Matplotlib «drawer» es:

[default]
circuit_drawer = mpl

Podemos usar cualquiera de los «backends» de visualización de circuitos válidos para esta configuración, esto incluye: text, mpl, latex, y latex_source.

Paso 5: Simular el Experimento¶

Qiskit Aer es un sistema de simulaciones de alto desempeño para circuitos cuánticos. Aer provee varios backends para obtener diferentes objetivos en las simulaciones.

Si encontramos problemas instalando Aer, podemos utilizar alternativamente el proveedor Basic Aer sustituyendo Aer con BasicAer. Basic Aer está incluido en Qiskit Terra.

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  BasicAer)
...

Para simular este circuito, usaremos qasm_simulator. Cada ejecución del este circuito producirá la cadena de bits 00 o 11.

simulator = Aer.get_backend('qasm_simulator')
job = execute(circuit, simulator, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 503, '11': 497}

Tal y como lo esperábamos, la cadena de bits obtenida es 00 aproximadamente el 50 por ciento de las veces. El número de veces que el circuito es ejecutado puede especificarse a través del argumento shots del método execute. El número de «shots» de la simulación anterior fue 1000 (el valor por defecto es 1024).

Una vez que obtengamos un objeto de tipo result, podemos acceder a los conteos a través del método get_counts(circuit). Esto nos da los conjuntos de resultados del experimento que ejecutamos.

Paso 6: Visualizar los Resultados¶

Qiskit proporciona muchas visualizaciones,

incluyendo la función plot_histogram para ver los resultados.

plot_histogram(counts)

Las probabilidades observadas \(Pr(00)\) y \(Pr(11)\) se calculan tomando los respectivos conteos y dividiendo por el número total de intentos (shots).

Nota

Intenta cambiar el número de shots en el método execute para ver cómo cambian las probabilidades estimadas.
