Mulai Bekerja dengan Qiskit¶

Alur kerja dalam menggunakan Qiskit terdiri dari tiga langkah umum:

  • Membuat: mendesain sebuah sirkuit kuantum yang merepresentasikan masalah yang sedang Anda tinjau.

  • Eksekusi: menjalankan eksperimen pada backends yang berbeda (yang mencakup sistem dan simulator).

  • Analisa: menghitung ringkasan statistik dan memvisualisasikan hasil eksperimen.

Berikut ini adalah contoh dari keseluruhan alur kerja, dengan setiap langkah akan dijelaskan secara rinci pada bagian berikutnya:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

# Use Aer's qasm_simulator
simulator = Aer.get_backend('qasm_simulator')

# Create a Quantum Circuit acting on the q register
circuit = QuantumCircuit(2, 2)

# Add a H gate on qubit 0
circuit.h(0)

# Add a CX (CNOT) gate on control qubit 0 and target qubit 1
circuit.cx(0, 1)

# Map the quantum measurement to the classical bits
circuit.measure([0,1], [0,1])

# Execute the circuit on the qasm simulator
job = execute(circuit, simulator, shots=1000)

# Grab results from the job
result = job.result()

# Returns counts
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)

# Draw the circuit
circuit.draw()

# Plot a histogram
plot_histogram(counts)

Langkah Alur Kerja¶

Program di atas dapat dibagi menjadi enam langkah:

  1. Impor paket

  2. Menginisialisasi variabel

  3. Menambah gerbang (kuantum)

  4. Visualisasi rangkaian sirkuit

  5. Mensimulasikan eksperimen

  6. Memvisualisasikan hasil

Langkah 1: Impor Paket¶

Elemen dasar yang dibutuhkan untuk program Anda dapat diimpor sebagai berikut:

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  Aer)
from qiskit.visualization import plot_histogram

Secara lebih rinci, yang diimpor adalah

  • QuantumCircuit: dapat dianggap sebagai instruksi dari sistem kuantum. Ini yang menampung semua operasi kuantum Anda.

  • execute: menjalankan/mengeksekusi rangkaian sirkuit/eksperimen Anda.

  • Aer: menangani backend simulator.

  • plot_histogram: membuat histogram.

Langkah 2: Menginisialisasi Variabel¶

Perhatikan baris kode berikut

circuit = QuantumCircuit(2, 2)

Di sini, anda melakukan inisialisasi dengan 2 qubit dalam keadaan nol; dengan 2 bit klasik yang diatur ke nilai nol dan circuit adalah sirkuit kuantum.

Sintaksis:

  • QuantumCircuit(int, int)

Langkah 3: Menambahkan Gerbang (Kuantum)¶

Anda dapat menambahkan gerbang (operasi) untuk memanipulasi register dari sirkuit Anda.

Perhatikan tiga baris kode berikut:

circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

Gerbang ditambahkan ke sirkuit secara satu-per-satu untuk membentuk keadaan Bell

\[|\psi\rangle = \left(|00\rangle+|11\rangle\right)/\sqrt{2}.\]

Kode di atas menerapkan gerbang berikut:

  • QuantumCircuit.h(0): Sebuah gerbang Hadamard \(H\) pada qubit 0, menempatkan qubit ini ke dalam keadaan superposisi.

  • QuantumCircuit.cx(0, 1): Sebuah operasi Not terkontrol (\(C_{X}\)) pada qubit kontrol 0 dan qubit target 1, menempatkan kedua qubit dalam keadaan Keadaan terkait.

  • QuantumCircuit.measure([0, 1], [0, 1]): jika Anda melewati seluruh register kuantum dan klasik untuk mengukur, hasil pengukuran qubit ke i akan disimpan dalam bit klasik ke i.

Langkah 4: Visualisasi Sirkuit¶

Anda dapat menggunakan QuantumCircuit.draw() untuk melihat sirkuit yang telah Anda rancang dalam berbagai bentuk yang telah digunakan dalam banyak buku teks dan artikel penelitian.

circuit.draw()

q_0: ┤ H ├──■──┤M├───
     └───┘┌─┴─┐└╥┘┌─┐
q_1: ─────┤ X ├─╫─┤M├
          └───┘ ║ └╥┘
c: 2/═══════════╩══╩═
                0  1

Dalam sirkuit ini, qubit-qubit disusun dengan qubit nol di bagian atas dan qubit satu di bagian bawah. Sirkuit ini dibaca dari kiri ke kanan, yang berarti bahwa gerbang yang diterapkan terlebih dahulu muncul lebih ke kiri.

Backend lazim untuk QuantumCircuit.draw() atau qiskit.visualization.circuit_drawer() adalah backend teks. Namun, tergantung pada environment lokal yang digunakan, Anda mungkin ingin mengubah ini menjadi sesuatu yang lebih cocok untuk kasus Anda. Hal ini dapat dilakukan dengan file konfigurasi pengguna. File konfigurasi pengguna ini seharusnya ada di dalam ~/.qiskit/settings.conf dan merupakan file .ini.

Sebagai contoh, sebuah settings.conf untuk pengaturan Matplotlib drawer adalah:

[default]
circuit_drawer = mpl

Anda dapat menggunakan salah satu dari backend pemvisualisasi sirkuit yang valid sebagai nilai untuk konfigurasi ini, ini termasuk text, mpl, latex, dan latex_source.

Langkah 5: Simulasi Eksperimen¶

Qiskit Aer adalah sebuah kerangka simulator performa tinggi untuk sirkuit kuantum. Qiskit Aer menyediakan beberapa backend untuk mencapai tujuan simulasi yang berbeda.

Jika Anda mendapati masalah dalam menginstal Aer, Anda dapat menggunakan penyedia Basic Aer sebagai alternatif dengan mengganti Aer dengan BasicAer. Basic Aer sudah termasuk dalam Qiskit Terra.

import numpy as np
from qiskit import(
  QuantumCircuit,
  execute,
  BasicAer)
...

Untuk mensimulasikan sirkuit ini, Anda akan menggunakan qasm_simulator. Setiap kali menjalankan sirkuit, sirkuit ini akan menghasilkan antara string bit 00 atau 11.

simulator = Aer.get_backend('qasm_simulator')
job = execute(circuit, simulator, shots=1000)
result = job.result()
counts = result.get_counts(circuit)
print("\nTotal count for 00 and 11 are:",counts)


Total count for 00 and 11 are: {'00': 527, '11': 473}

Seperti yang diharapkan, keluaran string bit adalah 00 kira-kira 50 persen dari total. Jumlah berapa kali sirkuit dijalankan dapat dispesifikasikan melalui argumen shots dari metode execute. Jumlah tembakan/shots dari simulasi ini diatur menjadi 1000 (pengaturan awalnya adalah 1024).

Setelah Anda memiliki objek result, Anda dapat mengakses counts dengan metode get_counts(circuit). Ini akan memberikan Anda hasil agregat dari percobaan yang Anda jalankan.

Langkah 6: Memvisualisasikan Hasil¶

Qiskit menyediakan berbagai macam visualisasi,

termasuk fungsi plot_histogram, untuk melihat hasil Anda.

plot_histogram(counts)

Probabilitas yang diamati \(Pr(00)\) dan \(Pr(11)\) dihitung dengan mengambil counts masing-masing dan dibagi dengan total jumlah tembakan/shots.

Catatan

Cobalah ubah argumen shots pada metode execute untuk melihat bagaimana estimasi probabilitas berubah.
