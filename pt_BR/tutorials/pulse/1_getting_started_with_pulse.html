Primeiros passos com OpenPulse¶

O Programa Qiskit Pulse oferece mais controle do que QuantumCircuit. Neste nível, você pode especificar a dinâmica de tempo exato dos sinais de entrada, em todos os canais de entrada. Grande parte do algoritmo quântico pode ser descrito com circuitos sozinhos usando operações de portão – geralmente é, quando você quer aplicar técnicas de mitigação de erros, ou, quando se torna útil usar o controle pulse-level para caracterizar uma qualidade de time-dependent de um sistema quântico.

Nesta página, vamos descobrir como construir e executar um programa Pulse básico que é chamado de schedule.

Inicializar¶

Importamos a classe Schedule, a partir do módulo qiskit.pulse. Para começar, vamos inicializar uma Schedule com o nome "getting_started".

[1]:

from qiskit.pulse import Schedule

sched = Schedule(name='getting_started')

Fácil! Temos uma schedule vazia agora.

Instruções de compilação¶

A próxima coisa que vamos querer fazer é criar algumas instruções para adicionar a nossa schedule.

Talvez, a instrução mais central para a construção de schedules seja a Waveform. É um sinal de pulso especificado como uma matriz de amplitudes complexas ordenadas ou amostras. Cada amostra é executada para um ciclo, um timestep dt, determinado pelo backend. Se quisermos saber a dinâmica em tempo real do nosso programa, precisamos saber o valor de dt. Por enquanto, vamos nos concentrar em como construir a instrução.

[2]:

from qiskit.pulse import Waveform

my_pulse = Waveform([0.00043, 0.0007 , 0.00112, 0.00175, 0.00272, 0.00414, 0.00622,
                        0.00919, 0.01337, 0.01916, 0.02702, 0.03751, 0.05127, 0.06899,
                        0.09139, 0.1192 , 0.15306, 0.19348, 0.24079, 0.29502, 0.35587,
                        0.4226 , 0.49407, 0.56867, 0.64439, 0.71887, 0.78952, 0.85368,
                        0.90873, 0.95234, 0.98258, 0.99805, 0.99805, 0.98258, 0.95234,
                        0.90873, 0.85368, 0.78952, 0.71887, 0.64439, 0.56867, 0.49407,
                        0.4226 , 0.35587, 0.29502, 0.24079, 0.19348, 0.15306, 0.1192 ,
                        0.09139, 0.06899, 0.05127, 0.03751, 0.02702, 0.01916, 0.01337,
                        0.00919, 0.00622, 0.00414, 0.00272, 0.00175, 0.00112, 0.0007 ,
                        0.00043],
                       name="short_gaussian_pulse")

Existem diversas maneiras de construir pulsos, sobre as quais você pode aprender em páginas posteriores. Desta vez, nós simplesmente passamos as amplitudes exatas do envelope de pulso que queremos executar como uma matriz. A matriz acima é uma função gaussiana avaliada em 64 pontos, com uma amplitude de 1 e um desvio-padrão de 8. A amostra (zero-indexed) \(i^{th}\) vai executar, a partir do tempo i*dt até (i + 1)*dt, modulada pela frequência do qubit. Pense nisso como um arbitrary waveform generator (AWG), executando as amostras fornecidas para o Waveform, misturado com um gerador de onda senoidal contínua produzindo um tom na frequência do qubit.

Os valores acima são reais, mas também podem ser complexos. A norma de amplitude de qualquer sinal de pulso está, arbitrariamente, limitada a 1. Cada sistema de backend, também pode impor mais restrições – por exemplo, um tamanho de pulso mínimo de 64. Saiba mais sobre como obter informações do backend ,aqui.

Instruções de schedule¶

Em seguida, temos que adicionar uma instrução para executar o sinal de pulso, que acabamos de construir. Isto significa especificar não, apenas o tempo que o pulso deve ser executado, mas também onde deve ser executado. Quando criamos circuitos, especificamos para qual qubit uma operação de portão deve ser aplicada. No Pulso, todo qubit tem múltiplos canais.

Vamos executar o nosso pulso no canal de unidade do qubit 0. O canal de unidade nos permite realizar operações individuais de qubit.

[3]:

from qiskit.pulse import Play, DriveChannel

qubit_idx = 0

sched = sched.insert(0, Play(my_pulse, DriveChannel(qubit_idx)))

Observe que o pulso que definimos opera no DriveChannel, que por sua vez é inicializado com o índice de qubit. Utilizamos Schedule.insert para iniciar o pulso no tempo t = 0.

Vamos rever o que fizemos, usando Schedule.draw:

[4]:

sched.draw(label=True)

[4]:

As formas em que as schedules podem ser compostas são detalhadas nesta página.

Este pulso dirigirá qubit 0. É modulado na freqüência ressonante do qubit 0, então conduzirá a transição do \(|0\rangle\) para o \(|1\rangle\). Não é calibrado para parar em um determinado estado, então, não saberemos que estado foi preparado, até olharmos para os resultados. Para nossa finalidade, não nos importamos com o estado que se apresenta.

O que falta fazer é adicionar uma medida. Existe uma função de utilidade conveniente para adicionar medições, mas requer dados do sistema de backend em que o programa será executado. Também precisaremos do backend para executar o programa.

Selecione um serviço¶

[5]:

from qiskit.test.mock import FakeAlmaden

backend = FakeAlmaden()

Adicione medições¶

Agora, podemos usar este serviço para adicionar as instruções de medição para nós.

[6]:

from qiskit.pulse.macros import measure_all

sched = sched.insert(sched.duration, measure_all(backend))

Vamos ver o que a função de conveniência adicionou, para nós, usando o desenho, novamente. Os pulsos de aquisição e medição são muito longos se comparados com nosso pulso inicial, então podemos usar o argumento plot_range` para prender a schedule.

[7]:

from qiskit.visualization import SchedStyle

sched.draw(plot_range=[0, 500], style=SchedStyle(axis_font_size=8))

[7]:

Há um novo pulso no MeasureChannel m0, nome abreviado para MeasureChannel(0). Este canal estimula leitura no qubit 0. Da mesma forma, a0 é a abreviação para AcquireChannel(0). A instrução Acquire no a0 indica, aos dispositivos de medição, quando começar a coletar dados de seus analog-to-digital converters (ADC) e por quanto tempo. Ele está configurado como um pulso de amplitude constante 1.

Execute¶

Nosso programa foi concluído! Não podemos usar nossa simulação de backend para executar programas, mas se você tiver uma conta IBM Quantum, você pode usar backend = IBMQ.load_account().get_backend(open_pulse=True) para ver se você tem um backend do OpenPulse ativado. Uma vez que você tenha um backend, podemos executá-lo da mesma forma que executamos circuitos:

job = execute(sched, backend)

É isso! Para obter os resultados, use result = job.result(). Você criou uma schedule do Pulso, contendo uma operação em qubit 0 seguida de uma medição, executou o experimento no backend e recuperou os resultados. Em seguida, cobriremos em detalhes as instruções diferentes que podem ser usadas em programas Pulse.
